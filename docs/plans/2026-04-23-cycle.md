# 사이클 (Cycle) 설계

상위 문서: [task-management.md](./2026-04-15-task-management.md)

---

## 배경

Linear의 cycle 개념을 도입한다. 기간별 목표를 설정하고 태스크를 배정하여 달성도를 측정한다.
나중에 ground-truth 목적 2 (변경 전후 비교)의 기반이 된다.

---

## 스키마

```sql
CREATE TABLE cycles (
    id           TEXT PRIMARY KEY,
    workspace_id TEXT NOT NULL REFERENCES workspaces(id),
    name         TEXT NOT NULL,
    status       TEXT NOT NULL,     -- planned, active, completed
    start_date   TEXT NOT NULL,     -- ISO 8601
    end_date     TEXT NOT NULL,     -- ISO 8601
    created_at   TEXT NOT NULL,     -- ISO 8601
    updated_at   TEXT NOT NULL      -- ISO 8601
);

CREATE TABLE cycle_tasks (
    cycle_id    TEXT NOT NULL REFERENCES cycles(id),
    task_id     TEXT NOT NULL REFERENCES tasks(id),
    assigned    TEXT NOT NULL,      -- planned (명시적 배정), auto (자동 포함)
    PRIMARY KEY (cycle_id, task_id)
);
```

### 설계 원칙

- cycle_events 테이블 불필요 — 상태가 단순(planned/active/completed)하고, 메트릭은 배정된 태스크에서 계산
- start_date/end_date는 계획된 일정 (엔티티 속성). 수정 가능
- goal 필드 없음 — 나중에 project(목표/기간이 있는 작업)와 겹침

---

## 상태

| 상태 | 의미 |
|---|---|
| planned | 생성됨, 아직 시작 안 함 |
| active | 현재 진행 중 |
| completed | 종료됨 |

### 제약

- 워크스페이스당 active cycle 1개
- planned는 여러 개 가능 (다음 사이클 미리 계획)

---

## 태스크 배정

### 명시적 배정

`cycle assign <cycle_id> <task_id>` — `assigned=planned`로 기록.

### 자동 포함 (기본 활성화)

active cycle이 있을 때:
- `task create` → 해당 cycle에 `assigned=planned`로 자동 추가
- `task move in_progress` → 미배정이면 `assigned=auto`로 자동 추가
- `task move done` → 미배정이면 `assigned=auto`로 자동 추가

---

## 사이클 리포트 집계

| 구분 | 조건 | 의미 |
|---|---|---|
| 계획 완료 | assigned=planned AND status=done | 계획대로 완료 |
| 계획 미완료 | assigned=planned AND status≠done | 계획했지만 못 끝냄 |
| 비계획 완료 | assigned=auto AND status=done | 계획에 없었지만 이번 사이클에 완료 |

## CLI/MCP 인터페이스

### CLI

```
seogi cycle create --workspace <name> --name "Sprint 1" --start 2026-05-01 --end 2026-05-14
seogi cycle list [--workspace <name>]
seogi cycle update <cycle_id> [--name "..."] [--start <date>] [--end <date>]
seogi cycle start <cycle_id>          # planned → active
seogi cycle complete <cycle_id>       # active → completed
seogi cycle assign <cycle_id> <task_id>
seogi cycle unassign <cycle_id> <task_id>
seogi cycle report <cycle_id>
```

### MCP 도구

`cycle_create`, `cycle_list`, `cycle_update`, `cycle_start`, `cycle_complete`, `cycle_assign`, `cycle_unassign`, `cycle_report`

## 결정 사항

- 용어: cycle (Linear 계승)
- 상태: planned, active, completed (cycle_events 불필요, status 직접 업데이트)
- 경계: 수동 시작/종료, start_date/end_date 명시 (수정 가능)
- 제약: 워크스페이스당 active cycle 1개, planned는 여러 개 가능
- 태스크 배정: 명시적(planned) + 자동(auto), 기본 활성화
- 리포트: `seogi cycle report <cycle_id>` 별도 명령어
- goal 필드 없음: 나중에 project와 겹침
