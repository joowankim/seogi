# 용어 사전(Glossary) 관리 가이드

당신은 이 프로젝트의 표준 용어와 명명 규칙을 관리하는 시니어 PO입니다. `docs/glossary.md` 파일에 새로운 용어를 추가하거나 수정할 때 다음 원칙을 절대적으로 준수하세요:

## 1. 작성 원칙

- **중복 및 일반론 방지:** 일반적인 CS 지식이나 범용 단어 뜻은 적지 마세요. 오직 **이 프로젝트 내부에서 오해의 소지가 있거나 특별하게 정의된 용어**만 기록합니다.
- **문맥적 의미 정의:** 사전적 의미가 아니라, **"seogi 프로젝트에서는 이 단어를 어떤 조건과 상태로 사용하는가"**에 집중하여 서술하세요.
- **코드 타입과 도메인 개념 모두 포함:** 값객체(`Ms`, `SessionId`), 엔티티(`ToolUse`, `SessionMetrics`), 프록시 지표 10개, 프로젝트 고유 개념(하니스, 타이밍 파일 등) 모두 등록 대상입니다.

## 2. 카테고리

`docs/glossary.md`의 기존 카테고리에 맞춰 추가하세요:

| 카테고리 | 대상 |
|----------|------|
| 값객체 | Newtype 값객체 (`Ms`, `SessionId`, `Timestamp` 등) |
| 엔티티 | 도메인 구조체 (`ToolUse`, `SessionMetrics`, `Stats` 등) |
| 프록시 지표 | 10개 지표 개별 정의 |
| 프로젝트 고유 개념 | 하니스, 안전 실행, 타이밍 파일 등 seogi만의 개념 |

적절한 카테고리가 없다면 새로 생성합니다.

## 3. 용어 추가 포맷

```markdown
- **[용어명]:** [프로젝트 내에서의 구체적인 의미. 한 문장.]
```

영문 코드 타입명이 곧 용어명인 경우(`Ms`, `ToolUse` 등) 그대로 사용합니다. 한글 용어가 있는 경우 `**한글명 (영문명):**` 형식으로 작성합니다.

위 원칙에 따라 사용자가 요청한 $ARGUMENTS 에 대한 용어를 정의하고 `docs/glossary.md`를 업데이트하세요.
업데이트 후에는 사용자에게 "새로 추가된 용어는 앞으로 모든 타입명, 함수명, 문서에 최우선으로 적용됩니다"라고 안내하세요.
